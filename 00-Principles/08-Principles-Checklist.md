# Principles Checklist — Venture Playbook OS

## 1. Purpose

This document provides a structured checklist to verify compliance with all principles defined in the Principles stage.

It is used as a gating mechanism before stage transitions and major decisions.

---

## 2. Principle Compliance Check

Before any stage transition or major decision, verify:

- [ ] All assumptions are explicitly declared
- [ ] All decisions are traceable
- [ ] All uncertainty is explicitly identified
- [ ] No hidden assumptions exist
- [ ] No stage boundaries are violated

---

## 3. Decision Quality Check

- [ ] At least two options considered (when applicable)
- [ ] Trade-offs documented
- [ ] Evidence supports decision
- [ ] Reversibility considered
- [ ] Complexity level considered

---

## 4. Validation Integrity Check

- [ ] Hypothesis defined
- [ ] Evidence generated or collected
- [ ] Signal vs opinion clearly separated
- [ ] Validation matches complexity level
- [ ] No premature conclusions drawn

---

## 5. Execution Quality Check

- [ ] Work decomposed into atomic tasks
- [ ] Outputs clearly defined
- [ ] Dependencies identified
- [ ] No hidden execution steps
- [ ] Output matches stage objective

---

## 6. AI Compliance Check

- [ ] AI operated within defined mode
- [ ] No silent assumptions made by AI
- [ ] AI outputs are structured and traceable
- [ ] AI did not finalize decisions
- [ ] AI respected stage boundaries

---

## 7. Complexity Compliance Check

- [ ] Complexity level explicitly defined
- [ ] Governance rigor matches complexity
- [ ] No uncontrolled complexity growth
- [ ] Complexity increases are justified
- [ ] System simplicity preserved where possible

---

## 8. Anti-Pattern Screening

- [ ] No premature solutioning
- [ ] No stage skipping
- [ ] No over-engineering
- [ ] No under-validation
- [ ] No decision without traceability

---

## 9. Stage Transition Gate

A stage is allowed to transition only if:

- [ ] All checklist sections are satisfied
- [ ] Exit criteria are fully met
- [ ] No unresolved high-risk uncertainty exists
- [ ] Human approval is explicitly granted

---

## 10. Final Principle

Compliance with principles is not optional.

It is a structural requirement for system correctness and venture reliability.
