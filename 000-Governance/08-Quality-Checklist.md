# Quality Checklist — Venture Playbook OS

## 1. Purpose

This checklist provides a standardized mechanism for evaluating the quality of documents, stage outputs, templates, decisions, and system artifacts.

Its purpose is to improve confidence before approval.

This checklist complements the review process.

---

## 2. Usage Rules

The checklist should be applied:

- before approval
- before stage transition
- before major decisions
- before publishing reusable artifacts

---

## 3. Evaluation Scale

Each item should be rated:

| Score | Meaning |
|---------|---------|
| 0 | Not addressed |
| 1 | Weak |
| 2 | Partial |
| 3 | Acceptable |
| 4 | Strong |
| 5 | Excellent |
